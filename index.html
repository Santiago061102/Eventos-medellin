📝 Primary Keys y Foreign Keys explicadas
PRIMARY KEY
La PK identifica de forma única cada fila. En este modelo cada tabla principal tiene su propia PK autoincremental.
EVENTOS: id_evento
LUGARES: id_lugar
ORGANIZADORES: id_organizador
ARTISTAS: id_artista
RESENAS: id_resena
EVENTO_ARTISTA: id
FOREIGN KEY
La FK enlaza una tabla con otra y evita referencias inválidas. Gracias a esto un evento no puede apuntar a un lugar o categoría inexistente.
eventos.id_categoria → categorias_evento.id_categoria
eventos.id_lugar → lugares.id_lugar
eventos.id_organizador → organizadores.id_organizador
evento_artista.id_evento → eventos.id_evento
evento_artista.id_artista → artistas.id_artista
resenas.id_evento → eventos.id_evento
